# Claude Instructions for Partycooler

## Keep documentation in sync

After every change to the codebase, update the following documents to reflect what changed:

- **`README.md`** — Update the implementation status table, project structure tree, and navigation table whenever a phase is completed, a new route is added, or a significant component is created.
- **`PLAN.md`** — Mark checklist items as complete (`[x]`) as each task is finished. Add notes under a phase if the implementation deviated from the original plan.
- **`DESIGN.md`** — Update if any design tokens, component patterns, or layout rules change during implementation.

This applies to all code changes — new files, refactors, and deletions.
